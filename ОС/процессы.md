процесс - является выполняющиеся программой 
что происходит при запуске программы:
- получение PID (process ID) 
- выделение памяти и ресурсов
- ядро отслеживает состояние процесса
### понятия
- PID - уникальный идентификатор процесса
- PPID - PID родительского процесса
- UID/GID ID группы(G - group) или пользователя(U - user) от имени которого был запущен процесс
- TTY - терминал с которым связан процесс
- STATE - текущее состояние процесса
- CMD - команда которая запустила процесс

#### команды
1. ps - показывает процессы
2. ps aux все процессы в системе
3. top/htop - мониторинг процессов в реал тайме
4. pstree - иерархия процессов
5. pgrep - найти процесс по имени
6. pidof - получить PID по имени
7. kill PID - убить процесс
8. kill -9 PID - принудительно завершить (SIGKILL)
9. nice/renice - управление приорететом процессов
10. jobs,bg,fg - управление фоновыми задачами
11. systemctl - управление системными службами демонами

-------
### жизненный цикл процесса
1. создание - с помощью системного вызова **`fork()`**
2. замена программы - через **`exec()`**
3. выполнение - процесс работает
4. завершение - **`exit()`**
5. ожидание родителем - **`wait()`**
----
### типы состояний 
- R - Running
- S - Sleeping (ожидает)
- D - Uninterruptible sleep (ожидает I/O)
- Z - Zombie (завершился но родитель не забрал статус)
- T - stopped (приостановлен)

----
### Демоны
- это процессы которые работают в фоне без терминала
- обычно стартуют при запуске через systemd
- примеры: sshd, nginx, docker, cron
##### приорететы
``` bash
nice -n 10 ./script.sh
```
- выдает приоретет при запуске от -20 до +19
```bash
renice -n -5 -p 1234
```
- renice меняет приоретет уже запущеного процесса (-p PID) (-n приоретет)

### где хранятся данные о процессах
Каждый процесс является директорией `/proc/PID/`
```bash
cat /proc/99/cmdline
cat /proc/99/status
cat /proc/99/environ
```
