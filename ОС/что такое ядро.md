ядро Linux - центральная часть [[ОС]] которая важна для понимания [[DevOps]]-инженеру

---

**Linux kernel** - работает между аппаратным обеспечением([[CPU]], [[RAM]], [[DISK]]) и пользовательскими процессами
ядро - мост между клиентской частью и аппаратной

---

### основные задачи ядра
ядро:
#### Process management
- как создавать процессы (`fork`,`exec`,`exit`)
- какой процесс получит процессорное время (планировщик - `scheldure`)
- как процессы взаимодействуют между собой (через сигналы, очереди, пайпы, сокеты)
- приорететы `nice`, `renice`
#### Memory Management
- выделяет и освобождает оперативную память для процессов
- управляет виртуальной памятью swap и кэш
- следит чтобы процессы не лезли друг на друга
#### Device Management
- содержит драйверы устройств, чтобы программы могли взаимодействовать с устройствами I/O
- взаимодейтсвует с устройствами через **/dev**
- предоставляет единый интерфейс для работы с любыми устройствами
#### File System Management
- обеспечение работы с файлами и каталогами
- поддерживает разные типы FS: ext4, btrfs, nfs
- реализует механизмы кэширования данных и контроля доступа.
#### Networking
- реализация и обработка [[TCP]]/[[IP]] стека
- управление сетевыми интерфейсами
- реализует [[NAT]],маршрутизацию,[[firewalls]]
#### additionally
- безопасность, права доступа, SELinux/appArmor, capabilities
- [[syscalls]] - интерфейс между программами и ядром
- Модули ядра можно расширять `.ko` без перезагрузки
```bash
insmod module.ko # вставить модуль
rmmod module.ko # удалить модуль
dmesg | tail # лог ядра
```
