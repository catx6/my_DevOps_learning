**Виртуализация** - технология позволяющая запускать несколько изолирвоанных систем([[ОС]]) на одном физическом сервере

Каждая **Виртуальная машина VM** имеет 
- свое ядро(виртуализированное)
- виртаульную оперативную память
- виртуальный диск
- виртуальную сетевую карту
Все делится между реальным железом

----

#### как все работает под капотом

За это все отвечает програма **гипервизор** (hypervisor)
он стоит между железом и виртуальными машинами

Виды гипервизоров

| Тип                | пример                         | применение                           |
| ------------------ | ------------------------------ | ------------------------------------ |
| тип **bare-metal** | VMware ESXi, KVM,Hyper-V, Xen  | на серверах, напрямую на железе      |
| тип **hosted**     | VirtualBox, VMware Workstation | на обычных ПК, в Linux/macOS/Windows |

##### KVM
KVM `Kernel-based Virtual Machine` 
- встроен в ядро Linux
- превращает сам Linux в гипервизор
- Управление через `virsh` или GUI `virt-manager`
Пример:
```bash
sudo apt install qemu-kvm libvirt-daemon-system virt-manager
sudo virt-install --name testvm --memory 2048 --vcpus 2 \
--disk size=10 --cdrom ubuntu.iso
```
-> создает виртуальную машину Ubuntu прямо из консоли

#### QEMU
- Эмулятор и гипервизор
- Может эмулировать даже архитектуры (x86, ARM)
- Часто используется в связке с KVM (ускоряет работу через аппаратную поддержку intel VT-x / AMD-V)

---

#### LXC / LXD
- легкая виртуализация - контейнеры
- работают на уровне ОС и используют общее ядро
- ближе к Docker но более системные
