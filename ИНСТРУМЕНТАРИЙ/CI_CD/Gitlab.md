gitlab - это опенсорс сервис для хранения и управления [[GIT]]-репозиториями
место где разработчики хранят код чтобы при работе над общим проектом разработчики могли вливать свои ветки с изменениями
и подтягивать чужие

#### отличия GitLab от Github
Github - облачный сервис где код хранится на облачных серверах майкрософта

GitLab - может работать как в облаке так и на собственном хосте (on-premise)

### CI/CD

GitLab имеет встроенный CI/CD инструмент, плюсом которого является возможность использования на собственном сервере

- GitLab Server - хранит, передает код ранерам
- GitLab Runner - исполняет работу описаную в пайплайне
- pipeline - последовательность действий для теста кода

что делать чтобы запустить на своем сервере:
1. установка GitLab
2. установка GitLab Runner 
3. зарегистрировать раннеры в GitLab

### пайплайны

1. создание внутри репозитория файла `gitlab-ci.yml`
каждое описаное действие в пайплайне называется job
для описания пайплайна используется декларативный стиль
```yml
run_job:
	script:
	 - pip install .
test_job:
	script:
	 - make test
	   
deploy_job:
	script:
	 - echo "Deploying..."
```
-> по-умолчанию все делается парарельно поэтому надо использовать stages:
```yml
stages:
	- test
	- deploy  
run_job:
	script:
	 - pip install .
test_job:
	script:
	 - make test
	   
deploy_job:
	script:
	 - echo "Deploying..."
```
