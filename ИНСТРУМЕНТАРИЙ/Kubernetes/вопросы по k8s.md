#### что такое kubernetes
[[kubernetes]] - это оркестратор контейнеров, который автоматизирует деплой, масштабирование, обновление и восстановление контейнеризированных приложений

----
#### Какую проблему решает kubernetes
- решает проблему управления десятками даже сотнями контейнеров
- автозапуск при падении 
- автоскейлинг
- балансировка трафика
- развертывание новых версий без простоя 
- распределение нагрузки между нодами

---

#### что такое minikube
Одноузловый локальный кластер Kubernetes, созданный для обучения и разработки. работает внутри VM или Docker

----
#### Отличие StatefulSet от Deployment
*Deployment* - ресурс kubernetes, предназначеный для развертывания приложения без сохранения состояния. при использовании PVC все реплики будут использовать один и тот же **том**, и ни один из них не будет иметь собственного состояния

*StatefulSet* - поддерживает состояния приложений за пределами жизненного цикла отдельных модулей **pod**, например для хранилища. Используется для приложений с отслеживанием состояния, каждая реплика модуля будет иметь собственное состояние и будет использовать собственный том

Также statefulset не перебросится на другую ноду. Есть диск, который привязан. И при вымирании ноды, он не переедет в отличие от деплоймента.

Sigterm statefulset может ждать какое-то время.

Statefulset для баз используется потому что там есть темплейт *PVC*.

PVC - **PersistentVolumeClaim**
- нужный размер 10G
- тип хранилища (StorageClass)
- режим доступа (ReadWriteOnce, ReadWriteMany)
- доп параметры (throttle, volumeMode)

|Deployment|StatefulSet|
|---|---|
|Stateless приложения|Stateful приложения|
|Pod-ы одинаковые|Pod-ы уникальные|
|Имена случайные|Имена фиксированные (pod-0, pod-1...)|
|PV может быть общим|У каждого pod свой PVC|
|Нет строгого порядка|Старт/удаление по порядку|

stateless - без состояния 
stateful - с состоянием хранит данные
