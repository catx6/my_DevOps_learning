Ветка - указатель на конкретный коммит

```bash
git branch -m oldname newname # переименовать ветку
git branch -d branch # удалить ветку
git branch -D branch # если не слита
```

---

#### merge
merge - сливает текущую ветку с указанной `git merge dev` 
[[merge]]


**merge** от **rebase**

|merge|rebase|
|---|---|
|сохраняет историю|переписывает историю|
|создаёт merge-коммит|делает историю линейной|
|безопасен|требует аккуратности|
|используется в командной работе|используется для чистой истории|

#### стратегии ветвления

GitFlow
- **main**
- **develop**
- **feature**
- **release**
- **hotfix**
Trunk-based
- одна ветка main
- короткие feature ветки (через день удаляются)
- быстрые merge
- CI/CD на каждый коммит
