jinja - шаблонизатор для Python. который позволяет создавать файлы с динамическим содержимым. это способ отделить логику приложения от представления, например HTML, конфигураций, скриптов

---
#### идея
- Вы пишете **шаблон** с переменными и логикой (например, циклы и условия).
- При выполнении программы вы передаёте данные в шаблон, и Jinja генерирует готовый текст с подставленными значениями.
```jinja
<!DOCTYPE html>
<html>
<body>
  <h1>Hello, {{ name }}!</h1>
  {% if logged_in %}
    <p>Welcome back!</p>
  {% else %}
    <p>Please log in.</p>
  {% endif %}
</body>
</html>
```
Python код для рендеринга:
```python
from jinja2 import Template

template = Template(open("hello.html").read())
rendered = template.render(name="Alice", logged_in=True)
print(rendered)
```
Результат
```html
<!DOCTYPE html>
<html>
<body>
  <h1>Hello, Alice!</h1>
  <p>Welcome back!</p>
</body>
</html>
```

---
#### Основные возможности

- **Переменные**: `{{ variable }}`
- **Условия**: `{% if condition %} ... {% endif %}`
- **Циклы**: `{% for item in list %} ... {% endfor %}`
- **Фильтры**: модификация данных, например `{{ name|upper }}` → `ALICE`
- **Макросы**: переиспользуемые куски шаблонов
- **Наследование шаблонов**: создаёте базовый шаблон и расширяете его в других файлах

---
#### Где используется
- **Веб-фреймворки**: Flask, Django (через шаблоны), FastAPI
- **Генерация конфигураций**: Ansible, SaltStack используют Jinja для динамических конфигов
- **Отчёты и скрипты**: создание текстовых файлов с данными
