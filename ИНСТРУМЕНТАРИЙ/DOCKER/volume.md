Docker volume (том) - механизм позволяющий сохранять данные вне контейнера

1. именнованные тома
	- данные хранятся в специальной папке `/var/docker/volumes`
	- управляется самим Docker
	- рекомендуется для большинства случаев
2. привязанные тома / bind mounts
	- позволяет привязать любую папку или файл хоста с контейнером
	- полезно при обновлении контейнера
3. tmps-тома
	- хранят данные только в оперативной памяти хоста
	- данные не хранятся и удаляются после остановки хоста
	- работают только на Linux
---

### создание контейнеров

`docker volume create my_volume` - создать volume **my_volume**
`docker volume ls` - показать список томов
`docker volume inspect my_volume` - посмотреть информацию о томе
`docker volume rm my_volume` - удалить можно только если не используется
