redis - это **in-memory key-value хранилище** тоесть данные лежат в оперативной памяти 

он может:
- хранить пары `ключ-значение`
- иметь TTl
- поддерживать сложные типы данных (списки, множества, хэши, JSON)
- работать как кэш, очередь, брокер сообщений или мини бд


### основные понятия Redis

|Команда|Что делает|Пример|
|---|---|---|
|`SET key value`|записывает значение|`SET user:1 "jenkins"`|
|`GET key`|получает значение|`GET user:1`|
|`DEL key`|удаляет|`DEL user:1`|
|`EXPIRE key 60`|время жизни 60 секунд|—|
|`INCR key`|увеличить числовое значение|—|
|`LPUSH list item`|добавить элемент в список|—|
|`PUBLISH` / `SUBSCRIBE`|pub/sub каналы|—|
## Как Redis работает технически

- Запускается как **демон** (`redis-server`).
- Данные хранятся **в оперативке**.
- Может сохранять дампы (RDB) или журнал изменений (AOF) на диск — для восстановления.
- Поддерживает **репликацию**, **кластеризацию** и **высокую доступность (Sentinel)**.
