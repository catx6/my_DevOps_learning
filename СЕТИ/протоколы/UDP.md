**UDP** - User Datagram Protocol протокол транспортного уровня но без **гарантии доставки**
используется где важна скорость

| Свойство                  | Описание                                                                      |
| ------------------------- | ----------------------------------------------------------------------------- |
| **Тип протокола**         | Без установления соединения (connectionless)                                  |
| **Надёжность**            | Не гарантирует доставку, порядок или отсутствие дубликатов                    |
| **Контроль ошибок**       | Есть только контрольная сумма (checksum), но нет механизма повторной передачи |
| **Скорость**              | Быстрее TCP, так как меньше накладных данных                                  |
| **Размер заголовка**      | 8 байт (TCP — минимум 20 байт)                                                |
| **Примеры использования** | Видеозвонки, онлайн-игры, стриминг, DNS-запросы                               |
## как работает UDP

1. данные разбиваются на датаграммы
2. каждая датаграмма отправляется отдельно **без соединения**
3. если часть датаграмм потеряется протокол не будет пересылать их заново

```python
import socket

# создаем UDP-сокет
sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# отправляем сообщение
sock.sendto(b"Hello UDP!", ("127.0.0.1", 9999))

# закрываем сокет
sock.close()
```
